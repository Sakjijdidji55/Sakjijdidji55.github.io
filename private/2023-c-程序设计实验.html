【A-7】DynamicArray

题目描述

设计通用的 C++ 类模板，实现动态数组DynamicArray类和动态的集合DynamicSet类，DynamicSet类继承DynamicArray类，动态数组具有动态增长和缩减的功能。

main函数已经写好，main函数的执行过程为先创建一个大小为n的数组，然后拷贝一个数组，再对原本的数组执行若干操作，然后从操作后的数组创建一个集合，最后对集合进行若干操作。

要求：类声明已经给出，实现以下两个类的成员函数即可。

  1. DynamicArray类

    1. 动态数组类，能够存储任意类型的数据；

    2. 实现以下公有成员函数：

      • DynamicArray()：默认构造函数，初始化一个空的动态数组；

      • DynamicArray(const DynamicArray& other)：拷贝构造函数，实现深拷贝；

      • ~DynamicArray()：析构函数，释放动态数组内存；

      • void push_back(const T& element)：在动态数组尾部添加元素，若达到容量则调整数组大小，进行扩容，每次扩容时将数组的当前容量翻倍；

      • void pop_back()：删除动态数组尾部的元素；

      • ```
        void insert(int index, const T& element);

        • 若index在范围0 <= index < _size内，删除下标index的元素；
        • 否则，抛出异常std::out_of_range("Index out of range")；

      • int find(const T& element);：查找数组中第一个等于element的元素，返回元素下标，找不到时返回-1；

      • virtual void add(const T& element)：虚函数，对于动态数组，在尾部添加元素（等价于push_back）；

      • virtual void remove(const T& element)：虚函数，对于动态数组，删除所有等于element的元素，不存在等于element的元素时什么都不做；

      • int size() const：返回动态数组的大小（已经实现）；

      • int capacity() const：返回数组的容量（已经实现）；

      • bool empty() const：检查动态数组是否为空（已经实现）；

      • T& operator[](int index)：重载 [] 运算符，用于访问动态数组中的元素，index超出数组范围时，抛出异常std::out_of_range("Index out of range")；

      • const T& operator[](int index) const：重载 [] 运算符的常量版本；

  2. DynamicSet类

    1. 动态集合类，继承DynamicArray类，集合中元素不重复，且按升序排序；
    2. 实现以下公有成员函数：
      • DynamicSet(const DynamicArray& other)：从数组中构造一个集合；
      • virtual void add(const T& element)：重写基类虚函数，添加元素到集合中，若集合中已经存在该元素，什么都不做；
      • virtual void remove(const T& element)：重写基类虚函数，从集合中删除等于element的元素，若集合中不存在该元素，什么都不做。

输入格式

  • 输入的第一行是一个正整数n，代表数组的长度；
  • 第二行是n个数，代表数组的元素；
  • 接下来是对数组的操作，包括多行，以单独的一行1结尾，中间每行代表对数组的一步操作，第一个字符代表要执行的函数，后面的数字代表函数的参数；
  • 最后是对集合的操作，以单独的一行1结尾，中间每行代表对集合的一步操作；
  • 操作符含义：
    • a：add；
    • r：remove；
    • f：find；
    • i：insert；
    • e：erase；
    • o：operator[]；
    • 集合操作只涉及add，remove和find。

输入样例

解释：数组有5个元素，对数组的操作是添加元素6，然后根据数组创建一个集合，对集合的操作是查找元素2。

提示：

  • 注意不能使用vector、algorithm等工具实现，"disable.h"是隐藏头文件，与本题解答无关，本题使用到的标准库头文件只有<iostream>和<stdexcept>；
  • 在模板类中调用基类的成员函数时要使用this指针；
  • 在提交的两个文件中注意分别include两个类声明的头文件，头文件开头使用#pragma once避免同一文件被多次编译；
  • 使用动态内存分配来管理动态数组的存储空间；
  • 在调整大小时，可以考虑每次扩容时将当前容量翻倍，以减少频繁的内存分配和释放操作；
  • 注意深拷贝的实现，以确保在拷贝构造和赋值操作时，动态数组的内容不会被共享；
  • 考虑异常安全性，确保在发生异常时不会造成资源泄漏。
